Historical Data Generator

Inserts synthetic data directly into the database for past dates. This populates Production Counts, Shift Reports (all report types), and Shift Summary with realistic data so you can test reporting, exports, and integrations without waiting for real production. Data is inserted via SQL, bypassing the live event pipeline.

Tip: For reports like Downtime, OEE, Operator Entry, and Andon Response to show data, enable "Generate process events" below. This creates state-transition events (fault/clear, clearToEnter/inCycle) that the report walkers analyze. Without events, only Production Counts and Shift Summary will have data.

Verification

Compares the exact values the generator inserted (ground truth) against what the production API returns when querying hourly counts. This validates that the full read pipeline (aggregation, style grouping, shift bucketing) produces correct results. Green = match, yellow = planned mismatch, red = actual count mismatch.

Note: Ground truth is held in memory from the last Generate run. If you restart the server, you'll need to regenerate before verifying. Verification only checks hourly_counts data, not process_events or shift_summary.
